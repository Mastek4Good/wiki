# Workshop Session: Mobile Wireframes

**Date:** Wednesday 24 June 2026

This session captured notes from mobile wireframes for a frontline case management experience.

The discussion focused on what outreach workers need to see and do quickly from a mobile device, especially when searching cases, reviewing priority information, and recording new interactions.

## Homepage - Mobile View

The homepage needs to use a simple layout.

Frontline staff need to quickly see all of their cases and access all active cases. They also need to be able to search across all cases in the system.

The homepage should surface the most important high-level information about each case, including:

* Name
* Gender
* Age
* At-a-glance risk information
* At-a-glance mental health challenges information

## Case Record - Mobile View

The case record needs to make recording an interaction one of the first and easiest actions available.

Frontline staff need to be able to quickly record an interaction while they are in the field, so the interaction capture entry point should be highly visible.

## Prioritised Case Information

Ania from Simon on the Streets identified the following priority information for the case record:

* Name
* Gender
* Date of birth
* Age
* Contact details
* National Insurance number
* Risk information
* Mental health challenges
* Link to ID, such as a passport scan

A photograph would be helpful for identification, but may not always be possible.

## AI Overview

The wireframe notes included the idea of an AI overview for quick reference.

This could help workers quickly understand the current context for a person before or during an interaction, but the notes do not yet define what information the overview should include or how it should be validated.

## Interaction History

Staff use the interaction history timeline very frequently.

The timeline needs to display the most important information first, so workers can quickly understand recent context without reading through every historical note.

## Search Within a Profile

Being able to search through a person's profile is essential.

It is not yet decided whether this should search only the interaction history or search more broadly across the full profile.

## Design Signals

The wireframe notes reinforce several design signals already present in the wider research:

* Mobile use must be treated as a primary workflow, not a cut-down secondary experience.
* Search and rapid access to active cases are central to the homepage.
* Interaction capture should be prominent and fast from the case record.
* Priority identity, contact, risk, and mental health information should be visible without forcing workers to dig through the record.
* Timeline design matters because workers rely heavily on interaction history.

## Open Questions

Questions to explore next include:

* What exact information should appear in the AI overview?
* How should the AI overview be checked for accuracy and safety?
* Should profile search cover only interaction history, or the full case record?
* How should the interface handle missing or uncertain identity information?
* When is it appropriate to show or capture a photograph?
* Which priority fields must be visible immediately on mobile, and which can sit behind progressive disclosure?

## Summary

The mobile wireframe notes point toward a simple mobile homepage for search and active case access, plus a case record that prioritises quick interaction capture, essential identity and risk information, interaction history, and profile search.
